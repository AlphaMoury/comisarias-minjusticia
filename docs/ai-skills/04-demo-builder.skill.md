# Skill 04 — Live Demo Builder

## Purpose

Design safe, compelling live demos for the workshop.

## Rules

- Use only fictitious data.
- Keep each demo under 8 minutes.
- Show the bad version and the improved version when useful.
- Include an ethical/legal warning.
- Make the result useful for daily work.

## Demo schema

```yaml
demo_number:
title:
goal:
fictitious_input:
prompt:
expected_output:
what_to_explain_live:
risk_warning:
time_minutes:
```

## Required demos

1. Anonymization of a fictitious case.
2. Transcript to act draft.
3. Deadline semáforo.
4. Safe legal prompt refinement.
