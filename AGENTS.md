# AGENTS.md — IA para Comisarías de Familia Deck

## Mission

Build a clear, visual, institutional presentation for a 2-hour workshop:

**IA para Comisarías de Familia: del uso informal al copiloto institucional seguro**

The audience is non-technical: family commissioners, legal teams, psychosocial teams, administrative staff, and public-sector stakeholders.

## Core message

AI does not replace legal judgment. AI helps organize, summarize, transcribe, review, prioritize, and draft — but always with secure data handling, traceability, and human review.

## Future message

Public officials will increasingly be able to create small applications, copilots, dashboards, workflows, and agents without knowing how to program. The key skill is translating institutional knowledge into instructions, rules, workflows, and safeguards.

## Inputs to read before acting

1. `docs/ai-context/brief_codex_ia_comisarias.md`
2. All files in `docs/ai-skills/`
3. `workflows/build-deck.workflow.md`
4. `examples/key-diagrams.mmd`

## Output requirements

Create a presentation with:

- 25 slides.
- Total duration: 120 minutes.
- Simple Spanish language.
- One central idea per slide.
- Large diagrams, minimal text.
- Notes for the speaker.
- Estimated time per slide.
- 4 live demos using fictitious data.
- No real case data.
- No legal hallucinations: if a legal reference is not provided, mark it as “verificar”.

## Visual style

- Clean, institutional, warm.
- Prefer diagrams over paragraphs.
- Use metaphors: semáforo, puente, cadena de valor, capas, copiloto, caja de herramientas.
- No overloaded dashboards.
- No dark cyberpunk style.
- No tech-bro aesthetic.
- Avoid tiny text.

## Diagram policy

For each key concept, produce:

1. A Mermaid version.
2. A plain-language description.
3. A suggested Excalidraw-style layout.
4. A slide-ready simplified version.

## Safety and privacy policy

Never include real names, case numbers, ID numbers, addresses, phone numbers, audio content, school names, neighborhood names, medical details, or family histories.

Use only fictitious or anonymized examples.

Always distinguish:

- Public information.
- Institutional internal information.
- Sensitive or reserved information.

## Human review policy

Every legal output must include a visible reminder:

“Borrador sujeto a revisión del funcionario competente.”

## Preferred workflow

1. Plan.
2. Draft slide outline.
3. Draft diagrams.
4. Draft demos.
5. Build presentation.
6. Run checklist.
7. Produce final artifacts.

Do not start coding before producing a concise implementation plan.
